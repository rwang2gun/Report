# ZZZ 스킬 태그 생성 스킬

## 목적

ZZZ 스킬 DB(`ZZZ 스킬 db01.csv`)의 태그 컬럼(부가효과, 적 디버프, 시너지)을
일관성 있게 유지하기 위한 태그 생성·정규화 작업을 수행한다.

스킬 설명 텍스트나 게임 내 정보를 입력받아 → DB 어휘에 맞는 표준 태그로 변환하는 것이 핵심이다.

---

## 위키 데이터 추출 (스크립트)

새 에이전트 데이터를 나무위키에서 추출할 때는 번들된 스크립트를 사용한다.

```bash
# 위치: scripts/scrape-wiki.js
node scripts/scrape-wiki.js <에이전트 풀네임>
# 예시
node scripts/scrape-wiki.js 호시미 미야비
```

**출력**: `output/<에이전트명>_wiki.json`
- `agent`: 풀네임, 속성, 클래스, 등급, 진영, 출시버전
- `skills`: 스킬별 `{ name, major, minor, description }`

스크립트 실행 후 JSON을 Claude에게 전달하면 아래 태깅 작업을 수행한다.
HTML 처리를 스크립트가 담당하므로 컨텍스트 토큰 소모가 대폭 줄어든다.

---

## 어휘집 참조 — 최우선 원칙

**반드시 먼저** `references/tag-vocabulary.md`를 읽어서 기존 태그 전체 목록을 숙지한다.

> **절대 규칙**: 태그는 반드시 어휘집에 있는 표현만 사용한다.
> 어휘집에 없는 자유형 텍스트(예: `그로기 수치 대량 누적`)를 절대 만들지 않는다.
> 가장 근접한 기존 태그를 선택하거나, 새 태그가 필요하면 사용자에게 먼저 확인한다.

기존 DB와의 일관성이 가장 중요한 원칙이다.

## 작업 유형별 처리 방법

### 1. 새 스킬 태그 생성

스킬 설명 텍스트(게임 내 스킬 설명, 위키 등)를 받아 태그를 생성할 때:

**처리 순서:**
1. 스킬의 `대분류`와 `소분류`를 먼저 파악한다 → 대분류별 전형적 패턴 참고
2. 텍스트에서 전투 판정 요소 추출 (무적, 패링, 가드 등)
3. 피해/버프 효과 추출 — **개인 버프와 파티 버프를 반드시 구분**
4. 전용 자원 관련 효과 추출 (전용 자원 소모 → `전용 자원` 컬럼, 획득 → 부가효과에 `X 획득`)
5. 적에게 부착/발생시키는 디버프·상태 추출 (`적 디버프` 컬럼)
6. 에이전트 고유 상태 진입·발동 효과 → 부가효과에 명시 (전용 버프 컬럼 없음)
7. 시너지 태그 해당 여부 판단 (파티 연계 효과인지)

**출력 형식:**
```
스킬명: [소분류: 스킬 이름]  (예: "궁극기: 잔설")
에이전트: [에이전트 DB의 간소화된 이름]  (예: "미야비")
대분류/소분류: [분류]

부가효과: [태그1, 태그2, ...]
전용 자원: [자원명] (에너지·데시벨·지원 포인트 제외, 에이전트 전용 자원만)
적 디버프: [태그1, ...] (타깃에게 부착하거나 발생시키는 상태)
시너지: [태그1, ...] (없으면 빈칸)

[태그 선택 근거 간략 설명]
```

### 2. 기존 DB 태그 정규화

`ZZZ 스킬 db01.csv`를 읽어서 비정규화된 태그를 표준화할 때:

**정규화 체크리스트 (순서대로 적용):**
1. **공백 불일치 수정** — vocabulary의 정규화 규칙표 적용 (`HP 회복` → `HP회복` 등)
2. **컬럼 앞뒤 공백 제거** — 시너지 컬럼 등에 선행 공백이 들어간 경우 strip 처리
3. **동의어 통일** — 어휘집의 "(위와 통일 권장)" 표시 항목 기준으로 통일
4. **영문 태그 처리** — 영문 태그는 한국어로 변환 (`Hunter's Gait` → `사냥꾼의 걸음`)
5. **의미 분리가 필요한 경우** — 통일 시 의미 손실이 있으면 사용자에게 먼저 확인
6. 수정 목록을 표(행 번호, 기존값, 수정값)로 정리 후 일괄 반영

### 3. 신규 에이전트 스킬 일괄 태깅

새 에이전트의 전체 스킬셋을 한 번에 처리할 때:

1. 에이전트의 속성, 클래스, 전용 자원, 고유 메커니즘 파악
2. `ZZZ 에이전트 db01.csv`에서 에이전트 간소화 이름 확인 (없으면 사용자 확인)
3. 전용 자원이 있으면 어휘집 전용 자원 테이블에 추가 필요 여부 확인
4. 에이전트 고유 상태/메커니즘 → 시너지 목록에 없으면 신규 태그 추가 제안
5. 스킬 목록을 대분류별로 묶어서 처리
6. 완성된 결과를 CSV 추가 형식으로 출력

## 태그 판단 기준

### 부가효과 판단
- **무적판정**: 회피, 돌격, 빠른지원 등 이동성 스킬에 대부분 붙음. 핵심스킬(추가능력·핵심패시브)에는 없음
- **높은그로기배율**: 패링지원, 일부 콤보공격. "강력한 그로기", "그로기 대량 누적" 표현 → `높은그로기배율` 사용
- **패링**: 패링지원 소분류에 필수. 일반 회피에는 붙지 않음
- **피해증가 계열**: `파티원피해증가`(파티 전체) vs `피해증가`(자신) 구분. 마찬가지로 이상 축적 효율 등 스탯 버프도 개인/파티 구분 필수
- **빠른지원**: "빠른 지원 연계" 또는 "빠른 지원 발동" 텍스트가 명시된 경우에만 붙임
- **피해유형**: 핵심스킬(추가능력·핵심패시브)은 직접 공격 피해가 없으면 빈칸으로 둠
- **에이전트 고유 상태 진입**: 전용 버프 컬럼 없음. "서리와 달 자세 진입", "폭발 상태 진입" 등 고유 상태 진입·발동 효과는 부가효과에 명시
- **전용 자원 획득**: 에이전트 전용 자원을 획득하는 효과는 `X 획득` 형태로 부가효과에 추가 (예: `내리는 서리 획득`)

### 전용 자원 판단
- **에너지, 데시벨, 지원 포인트는 기록하지 않음** (모든 에이전트 공통이므로)
- 에이전트 전용 자원(예: 내리는 서리, 서릿발 게이지, 섬락전압 등)만 `전용 자원` 컬럼에 기입
- 전용 자원을 소모하는 스킬 → `전용 자원` 컬럼에 자원명 기입
- 전용 자원을 획득하는 스킬 → `전용 자원` 컬럼 비우고, 부가효과에 `X 획득` 태그 추가
- 같은 스킬에서 소모+획득이 모두 있으면 컬럼에 자원명 기입 + 부가효과에 `X 획득`

### 적 디버프 판단
- 이 컬럼에는 **에이전트가 타깃에게 직접 부착하거나 발생시키는 상태·효과**만 기입
- 예: 빙염(적에게 부착), 서리열(적을 서리열 상태로), 정복(적에게 부여)
- 파티원이 발동하는 효과와 연동(반응)하는 메커니즘은 적 디버프가 아닌 시너지로 분류
- 어휘집 `적 디버프 태그 목록` 참조

### 시너지 판단
- **진영시너지**: 같은 진영 파티 편성 시 효과가 발동하는 경우
- **속성시너지**: 같은 속성 파티 편성 시 효과
- **에이전트 전용 메커니즘**: 에이전트의 고유 시스템명(예: `혜성 인력`, `서리와 달 자세`, `혼돈`)을 그대로 태그로 사용 — 어휘집의 에이전트 전용 시너지 목록 참고
- 핵심스킬(추가능력)에 주로 붙으며, 일반공격·회피스킬에는 거의 없음
- 어휘집에 없는 신규 에이전트 전용 메커니즘이면 새 태그로 추가하고 사용자에게 알림

## 새 태그 생성 원칙

기존 어휘에 없는 개념을 마주쳤을 때:

1. **유사 기존 태그 확인** — 의미가 90% 이상 같으면 기존 태그 사용
2. **계열 패턴 따르기** — `속성시너지-에테르` 같은 패턴이 있다면 동일 형식 적용
3. **한국어, 공백 없음 원칙** — 특수한 경우 제외하고 붙여쓰기
4. **사용자 확인** — 완전히 새로운 개념이면 제안 후 확인

## CSV 출력 형식

DB에 추가할 행을 CSV로 출력할 때:

```
이름,ZZZ 에이전트,대분류,소분류,피해유형,부가효과,전용 자원,적 디버프,시너지
```

- **이름** 컬럼 형식: `소분류: 스킬이름` (예: `궁극기: 잔설`, `핵심패시브: 서늘한 불꽃`)
- **ZZZ 에이전트**: `ZZZ 에이전트 db01.csv`의 `에이전트명` 컬럼 값과 정확히 일치 (간소화 이름 사용, 예: `미야비`)
- 부가효과가 여러 개이면 `"태그1, 태그2"` (쉼표+공백, 큰따옴표로 감쌈)
- 빈 항목은 빈 문자열

### 에이전트 DB 관련 규칙
- `ZZZ 에이전트 db01.csv`의 `에이전트명` 컬럼은 **간소화된 이름** (미야비, 루시 등)
- 신규 에이전트 추가 시 에이전트 DB에 `풀네임` 별도 컬럼을 두도록 권고 (예: 에이전트명=미야비, 풀네임=호시미 미야비)
